TOPIQ PROJECT DOCUMENTATION
===========================

Project Name:
Topiq - Smart Learning Resource Recommendation System

Project Type:
University Final Year Project

Project Summary:
Topiq is a Django-based web application that helps university students find useful study resources for academic topics. A student can search a topic such as "Deadlock", "Binary Tree", or "Normalization" and the system recommends the top video resources and reading resources. The project also includes student feedback, bookmark support, topic matching, recommendation scoring, and an AI study assistant for academic questions.

Main Folder Explanation:

manage.py:
Runs Django commands such as runserver, migrate, test, and createsuperuser.

topiq/:
Main Django project configuration. It contains settings, root URL routing, ASGI, and WSGI files.

website/:
Main Django application. It contains models, views, URLs, forms, templates, static files, recommendation logic, admin configuration, and seed data command.

website/models.py:
Defines the database schema for Semester, Subject, Topic, VideoResource, ReadingResource, and StudentInteraction.

website/views.py:
Handles homepage, search results, API search, AI chat, bookmark, and feedback requests.

website/recommender.py:
Contains the topic matching and recommendation logic. It uses TF-IDF and cosine similarity when scikit-learn is available, with a fallback string matching method.

website/templates/website/:
Contains HTML templates for the website layout, homepage, and search results page.

website/static/website/css/style.css:
Contains the full frontend design system, including dark theme, navbar, hero section, cards, result grid, About Us section, chat panel, and responsive styles.

website/static/website/js/main.js:
Contains frontend interactivity for search loading state, AI chat panel, feedback, bookmark, copy button, and toast notifications.

website/management/commands/seed_data.py:
Custom command for inserting demo semesters, subjects, topics, videos, and reading resources.

documentation/:
Contains all written project documents, reports, guide files, and presentation support files.

3. WEBSITE WORKFLOW
===================

3.1 Setup Workflow:

1. Open project folder.
2. Create or activate virtual environment.
3. Install dependencies from requirements.txt.
4. Configure .env values such as SECRET_KEY, DEBUG, ALLOWED_HOSTS, GROQ_API_KEY, or ANTHROPIC_API_KEY.
5. Run database migrations.
6. Run seed_data command for demo data.
7. Start Django server.
8. Open website in browser.

Common Commands:

source myenv/bin/activate
pip install -r requirements.txt
python manage.py migrate
python manage.py seed_data
python manage.py runserver


3.2 Page Workflow:

Homepage:
The homepage contains a fixed navbar, Topiq branding, hero search section, project highlight, feature cards, About Us section, and AI chat floating button.

Search Request:
When the student searches a topic, the browser sends:

GET /search/?q=topic-name

Django Routing:
topiq/urls.py sends website requests to website/urls.py.
website/urls.py maps /search/ to the search_results view.

Search Processing:
The search_results view validates the query, checks cache, calls search_topics(query), stores recent searches in session, prepares context, and renders results.html.

Recommendation Engine:
The recommender loads active topics from the database, builds a searchable topic corpus, normalizes the student query, compares the query with stored topics, selects the best matched topic, fetches top resources, and returns a structured result.

Results Page:
The results page displays top video suggestions, article/blog/PDF resources, metadata, difficulty badges, ratings, feedback buttons, bookmark button, average study time, and related topics.

Feedback Workflow:
When a student clicks Helpful or Not Helpful, JavaScript sends a POST request to /feedback/. The backend saves or updates StudentInteraction, changes helpful/not helpful counts, recalculates ml_score, and returns JSON to update the UI.

Bookmark Workflow:
When a student clicks Save Topic, JavaScript sends a POST request to /bookmark/. The backend stores the topic ID in the Django session and records a StudentInteraction.

AI Chat Workflow:
The AI chat panel sends academic questions to /api/chat/. The backend builds an academic-only prompt, calls the configured AI provider, and returns a concise answer to the chat panel.

6. RECOMMENDATION DESIGN SCHEMA
===============================

Topic Matching:

Input:
Student search query

Processing:
1. Normalize query by removing special characters and extra spaces.
2. Build topic corpus from topic name, subject name, tags, search keywords, and description.
3. Use TF-IDF vectorizer to convert text into numeric vectors.
4. Use cosine similarity to compare student query with each topic.
5. Select the highest matching topic above the similarity threshold.
6. If TF-IDF fails or scikit-learn is missing, use fallback string matching.

Output:
Matched topic, confidence score, top videos, top readings, average study time, and related topics.


Resource Ranking Formula:

normalized_view_count = view_count / 1,000,000
feedback_score = helpful_count / total_feedback
ml_score = rating * 0.4 + normalized_view_count * 0.3 + feedback_score * 0.3

Ranking Factors:

Rating:
Shows resource quality.

View Count:
Shows popularity and usage.

Student Feedback:
Shows real usefulness from Topiq users.

ML Score:
Final score used to order resources.
